What Ariana 1.1 includes

Version 1.1 updates the framework story from a first release candidate to a performance and package-safety release with a real admin-panel example.

Core

Compiled rendering path

Core bootstrapping now expects compiler/Vite generated render functions instead of a fallback runtime template evaluator.

Reactivity

Signals and computed state

Optimized dependency tracking, lazy computed values, keyed-list paths, and form-array correctness guards.

Vite

External HTML/CSS

Components use templateUrl and styleUrl with diagnostics, template checks, and generated render functions.

Forms

Typed form primitives

Signal-based controls, groups, arrays, validators, async validation, pending state, dirty and touched state.

Release

Security and package gates

Release gates include audit checks, tarball inspection, smoke tests, dry publish, and manual GitHub Actions publishing.

Example

Admin panel sample

The new example ports the earlier ITSurge admin design into an Ariana component with state, modal actions, search, and dashboard data.

Admin Pro example

The admin-panel example is based on the previous ITSurge Admin Pro structure: sidebar sections, topbar tools, cards, tables, badges, modal actions, theme controls, and dashboard analytics.

npm run build:admin
