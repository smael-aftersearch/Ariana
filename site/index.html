Framework pieces that prove the runtime.

The example is no longer just HTML/CSS. It exercises real Ariana primitives: DI providers at bootstrap, auth service, i18n service, query client, lazy route modules and signal-based UI state.

Components

Class-based screens

Components stay TypeScript-first with external templateUrl/styleUrl resources and no NgModule or standalone ceremony.

Routing

Lazy route modules

Admin pages are declared as lazy route definitions and imported on navigation, showing route module names and loading state.

Query

Visible loading states

Dashboard, users, orders, products, roles and reports are fetched through QueryClient with fake latency and cache status.

DI

Injected services

AuthService, I18nService, AdminApiService and QueryClient are registered at bootstrap and injected into the shell.

i18n

English and Persian

The admin can switch locale and direction, proving text and RTL/LTR layout behavior in a real dashboard.

Security

Release gates

1.1 includes audit checks, tarball inspection, smoke tests, dry publish and manual GitHub Actions publishing.

Admin Pro is the framework showcase.

The demo is inspired by the ITSurge Admin Pro design and now includes login, topbar tools, command palette, notifications, profile menu, theme palette, language switching, routed pages and data loading.

Run it

npm install
npm run demo:admin
npm run build:admin

Pages included

Dashboard, Analytics, Users, Roles, Products, Orders, Reports, Calendar and Settings. The shell lazy-loads route modules and then fetches data through the query client.

Release pipeline is part of the product.

Ariana 1.1 is prepared as a Performance & Security release candidate. Publish should happen from GitHub Actions after gates pass, not from a local terminal.

BuildAll packages and examples compile.
TestUnit and contract tests guard runtime behavior.
AuditStatic checks and dependency audit run before packing.
InspectTarballs are scanned before publish.
PublishManual GitHub Actions workflow controls npm release.
