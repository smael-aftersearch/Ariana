Ariana 1.0 · TypeScript-first UI framework

Class-based UI with signals and compiler-assisted templates.

Ariana gives you independent components, explicit rendering, external HTML/CSS, fine-grained reactivity, forms, routing, query state, rendering helpers, and a Vite plugin without Zone.js, NgModules, or a Virtual DOM.

// counter.page.ts
import { Component, computed, signal } from '@ariana-framework/core';

@Component({
  selector: 'ari-counter',
  templateUrl: './counter.page.html',
  styleUrl: './counter.page.css'
})
export class CounterPage {
  readonly count = signal(0);
  readonly double = computed(() => this.count() * 2);

  increment() {
    this.count.update(value => value + 1);
  }
}

What Ariana includes

Ariana 1.0 is a focused foundation for modern frontend applications: explicit, typed, small, and easy to reason about.

Core

Class-based components

Build screens as TypeScript classes with component metadata, external templates, and explicit methods.

Reactivity

Signals and computed state

Use signal values for precise UI updates and computed values for derived state.

Compiler

Template diagnostics

Catch invalid template syntax, unknown members, and type-aware template mistakes before runtime.

Routing

Routes and guards

Define route trees with params, data, providers, redirects, and guard chains.

Forms

Signal-based forms

Use controls, groups, arrays, sync validation, async validation, dirty/touched state, and pending state.

Data

Query and mutation state

Cache requests, dedupe in-flight fetches, retry failures, track stale data, and invalidate keys.

Templates and Vite

Use external HTML and CSS by default. The Vite plugin transforms resources, runs diagnostics, and can compile supported templates.

Forms that are just state

Forms are built on signals, so validation state, values, errors, and pending status are readable and explicit.

import { formControl, formGroup, required } from '@ariana-framework/forms';

const profile = formGroup({
  firstName: formControl('', [required()]),
  lastName: formControl('', [required()])
});

profile.patchValue({ firstName: 'Ariana' });
console.log(profile.valid());
  • ✓Controls expose value, valid, errors, pending, dirty, and touched signals.
  • ✓Groups aggregate child values and validation results.
  • ✓Arrays support repeated controls and dynamic form sections.
  • ✓Async validators are first-class and track pending state.

Performance position

Ariana 1.0 includes internal smoke benchmarks today. Cross-framework browser comparisons with Angular, React, Vue, Svelte, and Solid are documented as the next benchmark suite.

4.07mscore signal/computed updates sample
27.73msrouter repeated matching sample
12.07msquery cache operations sample
497.10msforms array operations sample and current watch area

Strong in v1

  • ✓Fine-grained signal and computed state updates.
  • ✓Explicit lifecycle cleanup for effects, listeners, and child rendering.
  • ✓Lightweight query cache and route matching in internal smoke checks.

Not claimed yet

Important: Ariana should not claim to be faster than Angular, React, Vue, Svelte, or Solid until repeatable browser benchmark results exist. The official benchmark plan covers counter updates, large table rendering, form-heavy screens, router stress, query cache stress, startup cost, and bundle cost.
