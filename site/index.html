A framework sample that proves runtime capability, not a static brochure.

The 1.1 admin showcase now demonstrates the exact primitives a senior frontend team would evaluate before trusting a framework.

Compiled UI

External HTML/CSS compiled by Vite

Components keep separate templates and styles while the runtime consumes compiled render functions.

Router

Lazy route chunks

Dashboard, Users, Orders, Reports and Settings are declared as lazy modules and imported on navigation.

Query

Real loading and cache status

Each data page fetches through QueryClient and shows a route-specific loading state only when data is missing.

DI

Provider-based services

AuthService, I18nService, AdminApiService and QueryClient are registered at bootstrap and injected into the shell.

i18n

English/Persian and RTL

The admin switches language and direction so layout behavior is visible, not just described.

Forms

Modal workflows

Entity modals, search, pagination, table state and page controls are wired into signal state.

Admin Pro is now a product-grade framework demo.

It includes login, routed shell, chunk inspector, query inspector, pagination, user/order/product/report pages, command palette, notifications, profile menu and theme controls.
