# Ariana 1.2.0-rc.1 Post-Release Guide

Use this guide after publishing `1.2.0-rc.1` with the `next` dist tag.

## Verify registry state

Run the GitHub workflow:

```txt
Actions -> Ariana 1.2 Verify NPM Release -> Run workflow
```

Use:

```txt
version: 1.2.0-rc.1
dist_tag: next
```

Or run locally:

```bash
RELEASE_VERSION=1.2.0-rc.1 NPM_DIST_TAG=next node scripts/verify-npm-release.mjs
```

Expected packages:

```txt
@ariana-framework/core@1.2.0-rc.1
@ariana-framework/compiler@1.2.0-rc.1
@ariana-framework/router@1.2.0-rc.1
@ariana-framework/forms@1.2.0-rc.1
@ariana-framework/query@1.2.0-rc.1
@ariana-framework/rendering@1.2.0-rc.1
@ariana-framework/vite-plugin@1.2.0-rc.1
```

Each package should have the `next` dist tag pointing to `1.2.0-rc.1`.

## Install smoke test

Create a temporary app outside the repo and install the release candidate:

```bash
npm init -y
npm install @ariana-framework/core@next @ariana-framework/router@next @ariana-framework/vite-plugin@next
```

Check that:

- packages install without peer dependency failures;
- package versions resolve to `1.2.0-rc.1`;
- imports resolve from `@ariana-framework/*` packages.

## Rollback strategy

Prefer dist-tag rollback over unpublish.

If `next` points to a bad RC and a previous RC exists, move the tag back:

```bash
npm dist-tag add @ariana-framework/core@<previous-version> next
npm dist-tag add @ariana-framework/compiler@<previous-version> next
npm dist-tag add @ariana-framework/router@<previous-version> next
npm dist-tag add @ariana-framework/forms@<previous-version> next
npm dist-tag add @ariana-framework/query@<previous-version> next
npm dist-tag add @ariana-framework/rendering@<previous-version> next
npm dist-tag add @ariana-framework/vite-plugin@<previous-version> next
```

If there is no previous RC, remove the `next` tag from the broken version only after confirming npm policy and package ownership requirements:

```bash
npm dist-tag rm @ariana-framework/core next
npm dist-tag rm @ariana-framework/compiler next
npm dist-tag rm @ariana-framework/router next
npm dist-tag rm @ariana-framework/forms next
npm dist-tag rm @ariana-framework/query next
npm dist-tag rm @ariana-framework/rendering next
npm dist-tag rm @ariana-framework/vite-plugin next
```

Avoid publishing `1.2.0-rc.1` as `latest`.

## Promotion to stable

Only promote to stable after:

- GitHub candidate workflow passes;
- npm dry run workflow passes;
- npm next release workflow passes;
- post-release verification passes;
- `/animate` manual test passes;
- `/route-outlet` manual test passes;
- one external install smoke test passes.

The stable version should be a separate release:

```txt
1.2.0
```
