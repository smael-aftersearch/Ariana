# Ariana 1.2.0-rc.1 Release Checklist

Use this checklist before publishing the release candidate.

## Version

```txt
1.2.0-rc.1
```

## Local verification

```bash
git pull origin main
npm install --no-audit --no-fund
npm run release:gates:v1.2
```

## Manual UI verification

```bash
npm run demo:admin -- --force
```

Open:

```txt
http://localhost:5173/animate
http://localhost:5173/route-outlet
```

Check `/animate`:

- Toggle hero.
- Toggle panel.
- Add and remove cards.
- Add and remove stagger items.
- Switch Flow, Data and Security.
- Compare Slow and Normal mode.

Check `/route-outlet`:

- Click Mount outlet.
- Switch Alpha, Beta and Gamma.
- Confirm the previous route leaves before the next route enters.
- Click Destroy and confirm the outlet clears safely.

## GitHub Actions candidate verification

Run:

```txt
Actions -> Ariana 1.2 RC Release Candidate -> Run workflow
```

Use:

```txt
version: 1.2.0-rc.1
run_gates: true
```

Download and inspect:

```txt
ariana-1.2.0-rc.1-npm-tarballs
ariana-1.2.0-rc.1-release-notes
```

## GitHub Actions npm dry run

Run:

```txt
Actions -> Ariana 1.2 NPM Dry Run -> Run workflow
```

Use:

```txt
version: 1.2.0-rc.1
dist_tag: next
```

Expected behavior:

- release gates pass;
- npm dry run succeeds;
- package tarballs are uploaded as artifacts;
- no real package is published.

## Local dry run verification

After release gates pass, run locally:

```bash
RELEASE_VERSION=1.2.0-rc.1 NPM_DIST_TAG=next npm run publish:v1:dry
```

Expected behavior:

- package tarballs are created in `npm-packages`;
- package names use `@ariana-framework/*`;
- internal package dependencies are rewritten to `1.2.0-rc.1`;
- npm runs in dry-run mode;
- no real package is published.

## Protected next release

Run only after candidate and dry-run workflows pass:

```txt
Actions -> Ariana 1.2 Next Release -> Run workflow
```

Use:

```txt
version: 1.2.0-rc.1
confirm: RELEASE_NEXT
```

Requirements:

- GitHub environment `npm-next` should require approval.
- Secret `NPM_TOKEN` must exist.
- Dist tag must remain `next`.

## Post-release verification

Run:

```txt
Actions -> Ariana 1.2 Verify NPM Release -> Run workflow
```

Use:

```txt
version: 1.2.0-rc.1
dist_tag: next
```

Or locally:

```bash
RELEASE_VERSION=1.2.0-rc.1 NPM_DIST_TAG=next node scripts/verify-npm-release.mjs
```

## Publish recommendation

Publish the release candidate using npm dist tag:

```txt
next
```

Recommended git tag:

```txt
v1.2.0-rc.1
```

Do not publish this candidate as `latest` until manual UI verification is complete.
