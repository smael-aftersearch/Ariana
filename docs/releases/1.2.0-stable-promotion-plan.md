# Ariana 1.2.0 Stable Promotion Plan

This plan starts only after `1.2.0-rc.1` has passed manual validation and feedback.

## Entry criteria

Do not start stable promotion until all of these are true:

- `1.2.0-rc.1` local gates passed.
- Candidate workflow passed with gates enabled.
- NPM dry run workflow passed with `next`.
- `/animate` manual validation passed.
- `/route-outlet` manual validation passed.
- GitHub draft release was created.
- RC was published to npm `next`.
- NPM registry verification passed.
- External install smoke from npm `next` passed.
- Feedback was collected and no blocking issue remains.

## Stable release changes

When entry criteria are complete:

1. Change root version from `1.2.0-rc.1` to `1.2.0`.
2. Create `docs/releases/1.2.0.md`.
3. Create or update stable checklist.
4. Run full release gates.
5. Build stable tarballs.
6. Run packed tarball content verification.
7. Run packed internal dependency verification.
8. Run packed install smoke.
9. Create stable manifest, checksum and summary.
10. Verify stable manifest.
11. Run npm dry run using `latest`.
12. Publish stable packages to npm `latest` only after approval.
13. Verify npm `latest` dist tag.
14. Publish final GitHub release.

## Version rule

Stable `1.2.0` must be rebuilt and re-verified.

Do not reuse the RC tarballs as the stable release artifact.

## Rollback rule

If a stable issue is found before publishing to `latest`, stop the promotion and create a new RC instead.

If an issue is found after publishing to `latest`, prefer moving the `latest` dist tag back to the previous stable version while preparing a patch release.
