# Ariana 1.2.0-rc.1 Release Notes

Ariana `1.2.0-rc.1` is the first release candidate for the animation and router-transition release line.

## Version

```txt
1.2.0-rc.1
```

## Main features

### SCSS and Sass `styleUrl` support

The Vite plugin now supports component `styleUrl` files ending in `.scss` and `.sass` through Vite inline style loading.

CSS files keep the existing direct inline read behavior.

### Template animation API

Compiled templates support static class-based animation attributes:

```html
<section animate.enter="fade-in" animate.leave="fade-out">
  Content
</section>
```

Supported in this release candidate:

- `animate.enter`
- `animate.leave`
- enter animation on initial render and mounted fragments
- leave animation for removed `@if` blocks
- leave animation for removed `@for` rows
- static class-name validation
- computed CSS duration cleanup
- bounded fallback cleanup

### Safer template compiler behavior

The Vite plugin now fails fast when template compilation cannot produce a compiled render function.

This prevents runtime-only failures where a component reaches bootstrap without a compiled `render` function.

The template control-block parser also ignores literal text such as `with @for.` and only treats `@if` and `@for` as control blocks when followed by a condition.

### Router transition metadata

Routes can define static transition metadata:

```ts
const routes = [
  {
    path: '/users',
    component: UsersPage,
    transition: {
      enter: 'route-enter route-rise',
      leave: 'route-leave route-drop'
    }
  }
];
```

`RouteMatch` exposes the resolved transition and nested child routes can inherit parent transition metadata.

### Router outlet coordinator

`@ariana/router` now exposes `createRouterOutlet`.

```ts
const router = createRouter(routes, '/');
const outlet = createRouterOutlet(router, '#route-host', {
  wrapperClass: 'route-view'
});

await outlet.render();
await outlet.navigate('/users');
```

The outlet coordinator:

- mounts matched route components into an isolated wrapper;
- runs leave transition on the previous wrapper;
- destroys the previous component after leave cleanup;
- replaces the host with the next wrapper;
- applies enter transition to the next wrapper;
- uses direct lightweight replacement when no transition exists.

### Core mount helper

`@ariana/core` now exposes `mountComponent` for reusable component mounting flows.

This is used by the router outlet so child route components can be destroyed without clearing the outlet host too early.

## Admin examples

### Dedicated animation playground

```txt
http://localhost:5173/animate
```

Includes:

- slow visible enter and leave animation;
- continuous orb animation;
- scene transitions;
- stagger list transitions;
- signal-driven card insertion/removal;
- performance notes: no polling, no observer, no eval.

### Router outlet transition demo

```txt
http://localhost:5173/route-outlet
```

Includes:

- real compiled route components;
- `createRouterOutlet` usage;
- Alpha/Beta/Gamma route transitions;
- manual outlet destroy test.
