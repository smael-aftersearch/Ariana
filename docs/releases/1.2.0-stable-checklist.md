# Ariana 1.2.0 Stable Release Checklist

Use this checklist only after `1.2.0-rc.1` has passed manual validation and feedback.

## Entry confirmation

- [ ] `docs/releases/1.2.0-rc.1-manual-validation.md` is completed.
- [ ] No blocking feedback remains from the RC.
- [ ] RC package was verified on npm `next`.
- [ ] External install smoke passed from npm `next`.
- [ ] Stable promotion plan was reviewed.

## Stable version preparation

- [ ] Change root version to `1.2.0`.
- [ ] Create `docs/releases/1.2.0.md`.
- [ ] Update release sequence for stable.
- [ ] Update package artifact naming expectations for `1.2.0`.

## Stable local validation

```bash
npm install --no-audit --no-fund
npm run release:gates:v1.2
npm run pack:v1.2:candidate
node scripts/verify-packed-tarball-contents.mjs
node scripts/verify-packed-internal-dependencies.mjs
node scripts/packed-candidate-smoke.mjs
node scripts/create-release-manifest.mjs
node scripts/verify-release-manifest.mjs
node scripts/create-release-summary.mjs
```

## Stable publish gates

- [ ] NPM dry run completed with `latest`.
- [ ] Stable artifacts were rebuilt from `1.2.0`.
- [ ] Stable manifest was verified.
- [ ] Stable summary was reviewed.
- [ ] Stable GitHub release draft was created.
- [ ] Approval was received for npm `latest`.

## Stable publish

- [ ] Publish `1.2.0` to npm `latest`.
- [ ] Verify every `@ariana-framework/*` package has `latest` pointing to `1.2.0`.
- [ ] Run external install smoke using `@latest`.
- [ ] Publish final GitHub release.

## Stop rule

If any stable validation step fails, stop promotion and create a new RC instead of publishing `latest`.
