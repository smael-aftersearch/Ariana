# Core Lifecycle Design

Ariana needs explicit lifecycle and cleanup rules before it can be used in larger apps.

## Goals

- predictable component creation
- predictable component destroy
- effect cleanup
- event listener cleanup
- parent-child cleanup order
- no global change detection

## Proposed lifecycle hooks

- `onInit()`
- `onDestroy()`
- `afterRender()`

These hooks are proposals and are not stable API yet.

## Cleanup rules

- effects created during component setup should be bound to the component lifetime
- DOM listeners generated by the compiler must be removed on destroy
- child components must be destroyed before parent cleanup completes
- failed component initialization must clean up partial resources

## Non-goals

- Angular-compatible lifecycle naming as a hard requirement
- Zone.js-like async tracking
- implicit global lifecycle registry

## 0.5.0 target

Ariana 0.5.0 should at least define the destroy contract and effect cleanup behavior.
