# Ariana 1.0 Release Readiness

Ariana 1.0 is prepared as a release candidate after the performance track, contract tests, security review, and release gates are complete.

## Release version

Target release version:

```text
1.0.0
```

The root package declares version `1.0.0`. Package candidate tarballs are generated with `RELEASE_VERSION=1.0.0` through the existing packaging scripts.

## Release-ready command

Run:

```bash
npm run release:ready:v1
```

This command runs the full release gate and a publish dry run.

## Required gates

The release must pass:

```bash
npm run build
npm test
npm run typecheck
npm run security:audit
npm run release:gates:v1
npm run publish:v1:dry
```

## Performance status

The performance result document is maintained at:

```text
docs/PERFORMANCE_RESULTS_V1.md
```

Current performance highlights to present:

- Ariana leads the `derived-counter` benchmark.
- Ariana leads the `keyed-list-1000-update` benchmark.
- Ariana remains competitive in list update and move scenarios.
- Form array performance must be verified after any forms change because a rejected validator-free fast path caused a regression and was reverted.

## Contract and safety status

Added or reinforced contract coverage for:

- lazy computed direct reads,
- computed chains,
- conditional computed dependencies,
- effect cleanup lifecycle,
- form array snapshot stability,
- form array move semantics,
- validator-free form control behavior.

## Security status

Security readiness now includes:

- `SECURITY.md` policy updated for the 1.x line,
- `npm run security:audit`,
- release gate integration for security audit,
- static repository scan for common secrets and unsafe APIs,
- conditional `npm audit --audit-level=high` when lockfiles exist.

## Packaging status

Candidate packages are generated by:

```bash
npm run pack:v1:candidate
```

The pack script stages package outputs, rewrites internal Ariana package imports to the release scope, applies `RELEASE_VERSION`, runs `npm pack`, and writes tarballs to `npm-packages`.

## Presentation checklist

Before presenting Ariana 1.0:

1. Pull the latest `main`.
2. Run `npm install` if dependencies changed.
3. Run `npm run release:ready:v1`.
4. Run `npm run bench:framework` and confirm `docs/PERFORMANCE_RESULTS_V1.md` is updated.
5. Review generated tarballs under `npm-packages`.
6. Present security, contract, performance, and packaging status together.

## Release decision

Ariana 1.0 is ready to present as a release candidate when all gates above pass and the final benchmark does not show a regression in the scenarios documented in `docs/PERFORMANCE_RESULTS_V1.md`.
